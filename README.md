## О проекте

**FXE / DevelNext 16.7** — это уникальная сборка **DevelNext**, основанная на официальном репозитории и дополненная новым функционалом.

Сборка сохраняет основу классического DevelNext, но расширяет IDE современными возможностями, обновлённым интерфейсом, улучшенным редактором кода, асинхронной загрузкой и более удобной работой с проектами.

Проект находится в активной разработке и постепенно дорабатывается под актуальные задачи 2026 года.

---

## Основная цель

Главная цель FXE / DevelNext — сохранить совместимость со старым DevelNext, но сделать IDE удобнее, современнее и стабильнее.

Основной упор сделан на:

- обновление интерфейса;
- улучшение конструктора форм;
- новые UI-возможности;
- ускорение работы с проектами;
- улучшение редактора кода;
- исправление старых проблем IDE;
- улучшение стабильности;
- подготовку DevelNext к использованию в 2026 году.

---

## Что нового

### Hot Reload

Добавлен `Hot Reload` для конструктора.

Теперь изменения в конструкторе применяются в запущенном приложении **в реальном времени**.

Можно менять:

- текст;
- размер;
- позицию;
- свойства объекта;
- CSS-класс;
- внешний вид элемента.

Результат сразу отображается в запущенном приложении без полного перезапуска.

---

### Множественный выбор объектов

Добавлена возможность выделять сразу несколько объектов на форме и редактировать их одновременно.

Теперь можно массово изменять:

- цвет;
- шрифт;
- размер;
- положение;
- видимость;
- доступность;
- CSS-класс;
- другие общие свойства объектов.

---

### Улучшенные `Ctrl+Z` / `Ctrl+Y`

Обновлена система истории действий в конструкторе.

Теперь действия корректно сохраняются в истории:

- перемещение объектов;
- изменение свойств;
- удаление элементов;
- изменение размера;
- массовое редактирование.

Действия можно отменить через `Ctrl+Z` или вернуть через `Ctrl+Y`.

---

### Инлайн-переименование

Файлы и папки в дереве проекта теперь можно переименовывать прямо на месте, без открытия дополнительных окон.

Это ускоряет работу с проектом и делает дерево файлов удобнее.

---

### Safe Rename

Добавлена функция безопасного переименования.

Например, если переименовать:

```txt
Button_Login
```

в:

```txt
Button_Exit
```

IDE предложит заменить это имя во всех связанных местах проекта.

---

### Новый Splash Screen

Добавлен новый универсальный `Splash Screen`.

Он стал:

- современнее;
- быстрее;
- аккуратнее;
- лучше подходит под обновлённый стиль FXE.

---

## Новые UI-компоненты

Добавлены новые компоненты для создания более современных интерфейсов:

- `Badge` — плашка для статусов, счётчиков и меток;
- `IconButton` — кнопка с иконкой;
- `SearchBox` — поле поиска с иконкой.

---

## Редактор и конструктор

В редакторе и конструкторе добавлены следующие улучшения:

- добавлено масштабирование рабочей области через `Ctrl + колесо мыши`;
- минимальный масштаб: `10%`;
- максимальный масштаб: `500%`;
- обновлён редактор кода;
- добавлен новый поиск и замена;
- нумерация строк теперь отображается как `1, 2, 3`, а не `001, 002, 003`;
- добавлено закрытие вкладок проекта через `Ctrl + W`;
- добавлена виртуальная PHP-структура в дереве файлов.

Виртуальная PHP-структура показывает под `.php` файлом:

- `class`;
- `method`;
- `property`.

---

## Интерфейс

В интерфейсе IDE были добавлены и изменены следующие элементы:

- добавлен новый нативный `titlebar` под Windows 10 и Windows 11;
- главное меню больше нельзя уменьшить до некорректного размера;
- добавлена новая оптимизированная тёмная тема;
- немного переработан общий UI интерфейса;
- `IDE Logging` теперь открывается во вкладке основной формы;
- добавлена подсветка логов;
- открытые проекты теперь отображаются в `MainForm`;
- добавлена поддержка SVG-иконок в дереве файлов.

---

## Дерево проекта

Дерево проекта было улучшено и очищено.

Добавлено:

- инлайн-переименование файлов и папок;
- поддержка SVG-иконок;
- виртуальная структура PHP-файлов;
- отображение классов, методов и свойств под `.php` файлом;
- защита важных файлов проекта от случайного удаления.

Очищены старые иконки дерева файлов:

```txt
org\develnext\jphp\gui\designer\editor\tree
```

## Проекты и загрузка

Добавлены улучшения для работы с проектами:

- добавлена индексация проекта;
- добавлен LSP-сервер;
- добавлена асинхронная загрузка IDE;
- асинхронно обрабатывается открытие проектов;
- асинхронно обрабатывается создание проектов;
- асинхронно обрабатывается Splash Screen;
- асинхронно обрабатывается закрытие IDE;
- при создании проекта добавлены новые папки;
- добавлена проверка `isProjectWillMigrate` по версии FXE из `.conf`.

---

## Безопасность проекта

Добавлены дополнительные проверки, чтобы случайно не повредить проект:

- заблокировано удаление корневой папки проекта;
- заблокировано переименование корневой папки проекта;
- заблокировано удаление файла `<project>.dnproject`;
- заблокировано переименование файла `<project>.dnproject`;
- добавлены проверки состояния проекта перед сборкой и запуском.

---

## Сборка и запуск проектов

Добавлены проверки, которые предотвращают некорректные действия при сборке и запуске:

- если проект открыт, сборка не запускается некорректно;
- если проект открыт и начинается сборка, приложение корректно закрывается;
- если уже идёт сборка, запуск проекта блокируется.

Это помогает избежать конфликтов между открытым проектом, сборкой и запуском приложения.

---

## Асинхронная загрузка

В IDE добавлена асинхронная обработка тяжёлых операций.

Теперь в фоне могут выполняться:

- открытие проекта;
- создание проекта;
- загрузка IDE;
- работа Splash Screen;
- закрытие IDE;
- отдельные операции подготовки проекта.

Это снижает вероятность зависаний интерфейса и делает работу IDE стабильнее.

---

## Логирование

Улучшена работа с логами IDE.

Добавлено:

- открытие `IDE Logging` во вкладке основной формы;
- подсветка логирования;
- очистка лишних предупреждений;
- перевод части шумных сообщений в более низкий уровень логирования.

Убраны лишние предупреждения:

```txt
WARN: AnchorPaneFormElement.xml not exists
```

```txt
WARN: slow class loading (preload CodeEditor/PhpAutoComplete)
```

```txt
WARN: IO in UI Thread порог, cannot be saved → DEBUG
```
